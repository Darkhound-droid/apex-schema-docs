# Usage Guide

## 1. Open the app

Page 1 introduces the project with a quick overview and links to the documentation generator.

---

## 2. Select objects (Page 2)

- Confirm your schema in the schema display at the top
- Use the **Tables**, **Views**, and **Packages** tabs to browse each object type
- Search by name using the search field on each tab
- Use checkboxes to include or exclude individual objects
- Use **Select All** and **Deselect All** buttons for bulk selection
- Pick your output format:
  - **Markdown** — best for pasting into Claude or ChatGPT
  - **JSON** — best for API-based LLM workflows
  - **Plain Text** — compact and token-efficient

---

## 3. Generate documentation

Click **Generate Documentation** to send your selected objects and chosen format to the output page.

You can also use the **Master Generator** to combine Tables, Views, and Packages into one single output document. Pass an empty string for any object type you want to exclude.

---

## 4. Review output (Page 3)

- Output is generated by `APEX_SCHEMA_DOCS_PKG` using Oracle's `USER_*` data dictionary views
- Check the **token estimate** to confirm your output fits within your LLM's context window
- Use **Copy to Clipboard** to paste directly into your LLM prompt
- Use **Download File** to save the output as:
  - `.md` for Markdown
  - `.json` for JSON
  - `.txt` for Plain Text
- Click **Regenerate** to go back and change your selection or format

---

## 5. About page (Page 4)

Contains project background, author details, version information, and the V3 and V4 roadmap.

---

## Tips

**For the best LLM results:** Use the Master Generator with all three object types selected and Markdown format. Paste the output as your first message with something like "Here is my Oracle schema. I will ask you questions about it next."

**Staying within context limits:** If your schema is large, select only the tables and packages relevant to the task at hand. The token estimate on the output page helps you gauge fit before copying.

**API workflows:** Use JSON format and save the output file. Include it as a system context document in your LLM API calls so every call in that project has full schema awareness automatically.
